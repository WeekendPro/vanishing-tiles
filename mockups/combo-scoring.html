Flat — today: every pick = 100, combo is cosmetic.
Linear — ×N = combo length (your ×5→500). Biggest variance; a perfect streak snowballs, one miss resets it.
Capped 5 — ×N = min(combo, 5). Keeps the thrill, tames runaway late-game streaks.
My take: put a persistent ×N chip right next to the score (it's the number it multiplies — cause and effect read instantly) and make the on-board burst the +points actually earned that pick, not "×N". The multiplier creates exactly the scoring variance you're after: a flawless run is worth multiples of a sloppy one. I'd start with Capped 5 (×N = min(combo, 5)) so a deep streak stays exciting without a single batch dwarfing the whole rest of the run — easy to lift the cap later if you want it wilder. Combo still ignites at 3 (chip hidden at ×1/×2, blooms at ×3).
